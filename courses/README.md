## 🏗️ How Each Course Folder Is Organized

Each course folder is intentionally structured to keep learning easy to review and easy to extend.

Typical contents include:

### 1. Course-Level README

A high-level overview of:

* what the course is about
* current progress
* what has been completed
* what the course section contains

### 2. Unit-Level Folders

The most important learning is broken down unit by unit, usually including:

* overview
* key concepts
* notes
* artifacts
* implementation details
* screenshots
* certificate placement where relevant

### 3. Implementation Artifacts

Where practical work exists, I include actual files used during the course, such as:

* Python files
* application files
* YAML configs
* templates
* workflow files
* MCP server/client code
* helper scripts

### 4. Screenshots and Evidence

Screenshots are included as proof of execution and to document important workflow moments such as:

* successful runs
* UI states
* server/client connectivity
* evaluation outputs
* automation results
* alerts and integrations

### 5. Notes and References

Notes are included to make the repository useful for:

* revision
* future reuse
* interview discussion
* explaining work to others
* turning course learning into project-ready knowledge
